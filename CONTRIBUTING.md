# Contributing to FocusGuard

Thank you for considering contributing to FocusGuard! Every contribution helps make this tool better for students, developers, and professionals worldwide.

## 🚀 Quick Start

1. **Fork** the repository
2. **Clone** your fork: `git clone https://github.com/YOUR_USERNAME/FocusGuard-Extension.git`
3. **Load** the `extension/` folder as an unpacked extension in `chrome://extensions/`
4. Make your changes and test thoroughly
5. **Submit** a Pull Request

## 📁 Project Overview

- **`extension/`** — The Chrome extension (vanilla JS, no build step)
- **`src/`** — The landing page (React + Vite + TypeScript)

## 🔧 Development

### Extension Development
The extension uses vanilla JavaScript with no build step. Edit files in `extension/` and reload the extension in Chrome to see changes.

### Landing Page Development
```bash
npm install
npm run dev
```

## 📝 Commit Convention

| Prefix | Purpose |
|--------|---------|
| `feat:` | New feature |
| `fix:` | Bug fix |
| `refactor:` | Code restructuring |
| `style:` | UI/CSS changes |
| `docs:` | Documentation |
| `perf:` | Performance improvement |
| `test:` | Tests |

## 💡 What to Contribute

- 🌐 Browser ports (Firefox, Edge, Safari)
- 🧪 Tests
- ♿ Accessibility improvements
- 📚 Documentation
- 🐛 Bug fixes
- ✨ New behavioral analysis features
- 📊 Dashboard visualizations

## ⚠️ Guidelines

- Follow existing code patterns and naming conventions
- Test in Chrome before submitting
- Keep PRs focused on a single change
- Write clear commit messages
- Update documentation if needed
